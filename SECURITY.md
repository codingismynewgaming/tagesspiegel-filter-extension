# Security Policy

## Reporting a Vulnerability

We take the security of our projects seriously. If you discover a security vulnerability, please report it privately via GitHub's private vulnerability reporting feature.

**Please do NOT open a public issue for security vulnerabilities.**

### How to Report

1. Go to the **Security** tab of this repository
2. Click on **Report a vulnerability**
3. Provide a detailed description of the vulnerability
4. Include steps to reproduce the issue if possible
5. Suggest potential fixes if you have any

### What to Expect

- **Initial Response**: Within 48 hours of your report
- **Status Updates**: Every 7 days while we investigate
- **Resolution Timeline**: Depends on severity, typically within 30 days

### Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

We recommend always using the latest version to ensure you have all security updates.

## Security Best Practices

When using this project:
- Keep your dependencies up to date
- Review the code before using in production
- Follow security best practices for your environment

Thank you for helping keep our projects secure! 🔒
